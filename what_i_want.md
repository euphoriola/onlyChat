
# 核心功能

使用代码，向API发送请求，获取API的流式回复，将对话过程展示在GUI

左侧一个边栏，显示会话列表
右边就是对话框，右上是历史对话，“我”和“AI”的聊天气泡交替出现；右下是输入框，可以输入多行的文本，然后有一个发送按钮

# 配置

各种配置我的想法是先写到config.json中，然后程序启动时读取加载
需要配置的内容:
 - API_KEY
 - BASE_URL
 - MODEL_NAME

还有哪些字段呢，比如，思考深度？，或者提供一些prompt模板？

# 我的想法

 - 根据模型供应商不同，应该给每个供应商创建一个独立的发送和接收请求的代码文件，因为每个供应商它的示例代码都有可能不同，比如deepseek和aliyun就应该是两个文件
 - 初始我们假设只有一个供应商，deepseek好了
 - 设计应该遵守less is more的原则，如果没有必要，不要添加任何额外的功能
 - 应该有一个功能，让用户可以添加prompt，并且在会话开始前选择prompt模板
